## Issues Found

### Minor: ServerNotFound ImportError subtype is unreachable

- **Severity**: minor
- **Location**: `/Users/glavatskikh.denis/StudioProjects/White-IPTV-KMP/shared/src/commonMain/kotlin/com/simplevideo/whiteiptv/feature/onboarding/mvi/ImportError.kt:9` and `/Users/glavatskikh.denis/StudioProjects/White-IPTV-KMP/shared/src/commonMain/kotlin/com/simplevideo/whiteiptv/feature/onboarding/OnboardingViewModel.kt:87-96`
- **Description**: `ImportError.ServerNotFound` is defined as a subtype of `ImportError` but `mapNetworkError()` never produces it. DNS resolution failures from the UseCase will be mapped to `ImportError.NoConnection` instead (the `else` branch catches everything not matching "timeout" or "HTTP"). The `resolveErrorMessage()` composable in `OnboardingScreen.kt` does handle it with `Res.string.error_server_not_found`, but the code path is dead.
- **Recommendation**: Either remove `ServerNotFound` from `ImportError` or add a pattern match in `mapNetworkError()` for DNS-related messages (e.g., messages containing "resolve", "DNS", or "UnknownHostException").

### Minor: mapNetworkError HTTP regex matches "HTTP" anywhere in message

- **Severity**: minor
- **Location**: `/Users/glavatskikh.denis/StudioProjects/White-IPTV-KMP/shared/src/commonMain/kotlin/com/simplevideo/whiteiptv/feature/onboarding/OnboardingViewModel.kt:91-93`
- **Description**: The `contains("HTTP")` check is case-sensitive and matches any message containing "HTTP" even if no status code follows. When the regex `HTTP (\d{3})` fails to find a match, it falls back to `ImportError.NoConnection`, which may be misleading for a message like "HTTP error without code". This is tested and the behavior is documented, but the fallback to NoConnection for an HTTP-related error is semantically questionable.
- **Recommendation**: Consider falling back to `ImportError.Unknown` instead of `ImportError.NoConnection` when "HTTP" is found in the message but no code can be extracted.

### Minor: CancellationException caught by runCatching

- **Severity**: minor (pre-existing)
- **Location**: `/Users/glavatskikh.denis/StudioProjects/White-IPTV-KMP/shared/src/commonMain/kotlin/com/simplevideo/whiteiptv/feature/onboarding/OnboardingViewModel.kt:62`
- **Description**: `runCatching` catches `CancellationException`, which can interfere with structured concurrency. If the coroutine is cancelled (e.g., ViewModel cleared during import), the cancellation will be swallowed and mapped to `ImportError.Unknown` instead of propagating. This is a pre-existing issue not introduced by the error-messages feature.
- **Recommendation**: Use `try/catch` with explicit exception types instead of `runCatching`, or re-throw `CancellationException` in the `onFailure` handler.
